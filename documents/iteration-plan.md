# Singo — 결정 로그 (Iteration Plan)

> ⚠️ **이 문서는 상태를 추적하지 않는다.** 다음 스토리·진행 상태·점수·소유는 **Tracker Boot가 진실의 원천**이다. 여기에 상태를 적으면 트래커와 이중화돼 매 Accept마다 어긋난다 — '이터레이션 플랜'이라는 이름이 흔히 빠지는 함정이 그것이다.
>
> **이 문서가 담는 것은 "왜"다.** 트래커에 안 남는 결정의 근거·맥락. 새 세션이 "왜 이 순서인가 / 무엇을 결정·기각했나"를 빠르게 복원하라고 둔다. "어디까지 했나"는 항상 트래커에서 잡는다(트랙 파일 완료목록도 stale할 수 있다).

**형식**: 시간 역순으로 "핵심 결정" 블록을 쌓는다. 각 블록 = 무엇에서 출발 → 무엇을 캐물음 → 무엇을 결정/기각 → 근거 → **다음 후속**.

> **최신 엔트리가 곧 핸드오프다.** 각 블록 끝에 **"다음 후속"** 포인터(후보 스토리 ID 1~3개 + 한 줄 이유)를 남긴다. 새 PM 세션은 `PM 시작` 시 이 **최신 엔트리의 "다음 후속" + current iteration**만 보고 다음 스토리를 잡는다.
